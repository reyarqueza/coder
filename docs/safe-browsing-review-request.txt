Google Search Console — Safe Browsing Review Request
====================================================

Property: bahamuto.com (domain property)
Affected URLs: https://coder.bahamuto.com/ and OAuth callback paths

Threat type reported (Transparency Report):
- Social engineering: "Try to trick visitors into sharing personal info or downloading software"

Steps taken to verify and remediate
-----------------------------------

1. Confirmed root cause scope
   - bahamuto.com is flagged in Google Safe Browsing Transparency Report
   - coder.bahamuto.com inherits parent-domain reputation; Chrome blocks OAuth callback returns
   - DNS for coder.bahamuto.com resolves correctly (Cloudflare → Vercel)

2. Dependency audit (package.json / package-lock.json)
   - No typosquatted packages detected
   - No suspicious install scripts beyond standard sharp/unrs-resolver
   - npm audit: 2 moderate advisories (postcss transitive via next); no malware indicators

3. Source code audit
   - Zero uses of dangerouslySetInnerHTML, eval(), new Function(), or dynamic script injection in app source
   - No third-party <script> tags; fonts loaded via next/font (self-hosted)
   - Login page uses server-side NextAuth GitHub OAuth only — no password/credential collection
   - Added explicit domain context on login page: "coder.bahamuto.com" and redirect notice

4. Public assets audit (/public)
   - Six SVG files only; no rogue HTML or JavaScript

5. Production verification
   - /login serves expected Next.js app content with no external script sources
   - GitHub OAuth redirect_uri correctly set to https://coder.bahamuto.com/api/auth/callback/github
   - Legal pages present: /privacy, /terms, /cookies

6. Infrastructure
   - Hosted on Vercel; DNS managed in Cloudflare
   - No evidence of DNS hijack or unauthorized deployments in repository

Conclusion
----------
This is a legitimate educational coding practice app (Coder Bahamuto). Authentication is
standard NextAuth v5 with GitHub OAuth. The site does not harvest credentials. We believe
the social-engineering classification is a false positive, likely triggered by OAuth callback
URL patterns on a subdomain of a newly registered domain.

Please re-evaluate bahamuto.com and coder.bahamuto.com for delisting.

Submission checklist
--------------------
1. Open https://search.google.com/search-console
2. Select bahamuto.com property (verify via existing TXT record if needed)
3. Go to Security & Manual Actions → Security issues
4. Mark issues as fixed
5. Paste the "Steps taken" section above into the review request
6. Submit review — typical turnaround 1–3 days

After delisting
---------------
1. Test in Chrome incognito: https://coder.bahamuto.com/login
2. Complete GitHub OAuth and confirm redirect to /dashboard
3. Verify session cookie is set
